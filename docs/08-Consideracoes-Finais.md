# Considerações Finais

## Reflexões sobre o Desenvolvimento

<p align="justify">O desenvolvimento deste projeto proporcionou uma experiência prática relevante na construção de aplicações backend, permitindo a aplicação integrada de conceitos fundamentais de engenharia de software.</p>

<p align="justify">Ao longo do processo, foram consolidados conhecimentos relacionados à criação de APIs RESTful com ASP.NET Core, modelagem de banco de dados relacional e utilização do Entity Framework Core como ferramenta de persistência de dados. A implementação das entidades, seus relacionamentos e as regras de negócio contribuíram diretamente para a compreensão de cenários próximos a sistemas reais.</p>

<p align="justify">Também foi possível reforçar a importância de boas práticas, como organização do código, separação de responsabilidades e validação de dados, fatores essenciais para garantir a manutenção, escalabilidade e confiabilidade da aplicação.</p>

<p align="justify">Além disso, o processo de testes e ajustes permitiu o desenvolvimento de uma abordagem mais analítica na identificação e resolução de problemas, simulando situações comuns no desenvolvimento de software.</p>

---

## Sugestões de Melhorias

Apesar de atender aos requisitos propostos, o sistema pode evoluir com a inclusão de novas funcionalidades e melhorias, tais como:

- Implementação de autenticação e autorização (ex: JWT);
- Criação de perfis de usuário com diferentes níveis de acesso;
- Desenvolvimento de uma interface front-end para consumo da API;
- Implementação de paginação e filtros mais avançados nas consultas;
- Padronização das respostas da API e tratamento global de erros;
- Implementação de logs e monitoramento da aplicação;
- Deploy da aplicação em ambiente de nuvem.

Essas melhorias tornariam a aplicação mais robusta, segura e alinhada com padrões utilizados em ambientes de produção.

---

## Considerações Gerais

<p align="justify">O projeto demonstra a capacidade de desenvolvimento de uma API estruturada, com regras de negócio bem definidas e integração com banco de dados relacional.</p>

<p align="justify">De forma geral, a aplicação cumpre seu objetivo de simular um sistema de locadora de veículos, oferecendo funcionalidades essenciais para gerenciamento de dados e operações, além de servir como base sólida para futuras evoluções.</p>
