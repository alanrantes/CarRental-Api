# Introdução

<p align="justify">Este projeto consiste no desenvolvimento de uma API RESTful para gerenciamento de uma locadora de veículos, com foco na modelagem eficiente de dados e na construção de um backend escalável e bem estruturado.</p>

<p align="justify">A solução foi idealizada com base em cenários reais do mercado, nos quais é necessário controlar informações relacionadas a veículos, clientes, fabricantes e operações de locação. O projeto busca demonstrar, de forma prática, a aplicação de conceitos fundamentais de engenharia de software, como modelagem de banco de dados, organização em camadas e integração com sistemas de persistência.</p>

---

## Objetivos do Sistema

O sistema tem como principal objetivo fornecer uma base sólida para o gerenciamento das operações de uma locadora de veículos, contemplando:

- Cadastro e gerenciamento de fabricantes;
- Classificação de veículos por categoria;
- Controle de veículos disponíveis para locação;
- Gerenciamento de dados de clientes;
- Registro completo de operações de aluguel, incluindo período, valores e quilometragem;
- Garantia de integridade e consistência dos dados por meio de relacionamentos entre entidades.

Além disso, o projeto visa consolidar o uso de boas práticas no desenvolvimento de APIs e na utilização de frameworks modernos para acesso a dados.

---

## Visão Geral do Sistema

<p align="justify">A aplicação foi desenvolvida como uma API RESTful utilizando o ecossistema .NET, permitindo a exposição de endpoints para operações de CRUD (Create, Read, Update, Delete) e consultas mais complexas envolvendo múltiplas entidades.</p>

A arquitetura adotada promove separação de responsabilidades, facilitando a manutenção, escalabilidade e organização do código.

As principais tecnologias utilizadas foram:

- C#
- ASP.NET Core Web API
- Entity Framework Core (ORM)
- SQL Server Express
- Swagger (OpenAPI) para documentação e testes das APIs

<p align="justify">O sistema foi estruturado para refletir boas práticas de desenvolvimento backend, incluindo o uso de ORM para mapeamento objeto-relacional, validação de dados e organização modular do projeto.</p>
